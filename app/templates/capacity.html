📤 Historische Daten importieren

Der Import unterstützt zwei verschiedene Datenkategorien, die in der Datenbank getrennt gespeichert werden. Wählen Sie den passenden Tab für Ihre Importdatei.

Importiert Kapazitätsdaten für physische Speichersysteme (Block, File, Object, Archiv, Backup). Jede Zeile entspricht einer Tages-Messung eines im Dashboard konfigurierten Systems. Bestehende Einträge für dieselbe Kombination aus Datum und System werden überschrieben.

CSV-Format – Physische Systeme:

date,system_name,total_tb,used_tb,free_tb,percent_used 2024-01-08,mein-system-01,100.00,45.50,54.50,45.5 2024-01-15,mein-system-01,100.00,47.20,52.80,47.2

• date: ISO-Datum (YYYY-MM-DD)
• system_name: exakter Name des Speichersystems (wie im Dashboard konfiguriert)
• total_tb / used_tb / free_tb: Kapazitäten in TB
• percent_used: Auslastung in Prozent (0–100)

📁 CSV-Datei hierher ziehen oder klicken zum Auswählen

Importiert historische Daten für Storage on Demand (Pure1 / Evergreen One) direkt über die Pure1 REST API. Die Daten werden je Lizenz und Tag aus dem Metrics History-Endpunkt von Pure1 abgerufen und in der Datenbank gespeichert. Sie sind damit unabhängig vom wöchentlichen Live-Cache und erlauben eine vollständige Rückerfassung historischer Werte.

Pure1 API speichert Tageswerte üblicherweise für ca. 2 Jahre zurück. Für ältere Zeiträume ist kein direkter Abruf möglich.
